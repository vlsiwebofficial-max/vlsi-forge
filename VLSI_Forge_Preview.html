Release Notes

What Changed in This Update

9 fixes across security, correctness, performance, and VLSI-specific tooling

🔴 Critical

Dangerous Code Detection

Submitted Verilog is now scanned for $system(), write-mode $fopen, and absolute path includes before any compilation starts.

🔴 Critical

VCD Stored in MongoDB

Waveform data is now base64-encoded and persisted in the database — no more broken download links after server restart.

🔴 Critical

Non-Blocking Simulation

Replaced subprocess.run() with asyncio.create_subprocess_exec. The event loop no longer freezes during 10-second simulations.

🔴 Critical

Stats N+1 Query Fixed

User stats now use a single MongoDB aggregation pipeline instead of one DB call per passed submission.

🔴 Critical

Route Conflict Fixed

/submissions/user/me is now registered before /submissions/{id} — the dedicated endpoint was previously shadowed.

🟡 High

Rate Limiting

Submission endpoint is now capped at 10 requests per minute per IP using slowapi. Prevents compute abuse.

🟡 High

MongoDB Indexes

8 indexes created on startup including a TTL index on sessions. Lookup performance improved significantly on large collections.

🟡 High

Smart Output Comparison

Output normalization now handles whitespace, hex literals (8'hFF → 255), and comment lines — no more false failures.

🟢 New Feature

In-Browser Waveform Viewer

VCD files are parsed into JSON and rendered interactively in the browser. No more downloading to GTKWave.

🟢 New Feature

Verilator Lint Integration

After every simulation, design code is linted with verilator --lint-only -Wall. Warnings are shown in a new "Lint" tab.

🟢 New Feature

SystemVerilog Support

Language selector added to the editor. SV files are compiled with iverilog -g2012 to support always_ff, logic, interfaces, and more.

🟢 New Feature

Admin Pagination

Admin endpoints now support ?skip=&limit= with a hard cap — no more loading 10,000 users into memory.

Security Improvement

Dangerous Code Detection

Code is scanned for dangerous Verilog tasks before any compilation happens.

Submission #1 — Safe code

module half_adder( input a, b, output sum, carry ); assign sum = a ^ b; assign carry = a & b; endmodule

Correctness Fix

Smart Output Normalization

The same correct answer no longer fails due to formatting differences.

❌ Before (raw string compare)

// Expected: sum = 15 carry = 0 // Actual output: sum = 4'hF carry = 1'b0 // Result: FAIL ← (wrong!) // "15" ≠ "4'hF" — same value!

✓ After (normalized compare)

// Expected (normalized): sum = 15 carry = 0 // Actual (normalized): sum = 15 carry = 0 // Result: PASS ✓ // Hex → decimal, whitespace trimmed
